SSUCF = Social Science Using the Chronoflux Framework.

The five Chronoflux constructs (hydrodynamic core, array order):
  1. Continuum (ρt) — baseline cohesion transport

  • Blank σ / Iτ / Jμ → Grok observational inference relative to the ω question.

3.1 APPLICATION SHELL AND NAVIGATION
--------------------------------------------------------------------------------

Evolve is a Flutter app (Windows, Android, web on GitHub Pages) with five main tabs:

  1. Analysis    — percent chance and social cohesion scenario engine
  2. Wallet      — Perccent (PERC) registration, balance, send/receive, chain sync
  3. Voting      — Full Community Governance (FCG) parish council voting
  4. Credit      — attribution, FCG link, governance goals
  5. (Ward)      — Community Ward Voting dapp inside Wallet suite

On first launch (unsigned user): splash → wallet registration gate → main app.
Wallet sign-in is required before analysis faucet credits PERC or ward votes are cast.

Region focus: UK, USA, Europe — affects base rates, registry matching, and localized
output strings. Language: English, Spanish, French, German, Portuguese, Arabic,
Chinese, Hindi, Japanese.


--------------------------------------------------------------------------------
3.2 ANALYSIS TAB — PERCENT CHANCE MODE
--------------------------------------------------------------------------------

User enters:
  • Posed question (ω / vortex) — required
  • Optional topic, source URL, construct fields (σ, Iτ, Jμ)
  • Region selector

Pipeline (EvolveEngine.analyze, mode = percentChance):

  1. InputParser.enrich — normalises fields, infers missing constructs
  2. OutcomeFeasibilityChecker — foreclosed outcomes cap PART TWO
  3. Hydrodynamic core — SCS per construct from field text + semantics
  4. PART ONE — baseline progressive/regressive split, baseline SCS
  5. PART TWO — refined channels, elite factor, refined SCS
  6. PART THREE — three actionable lever sentences (agent + hooks)
  7. Percent chance — continuum equation (regressive × 0.55 + σ × 0.25 + strain × 0.2)
  8. ForecastCalibrator — blends heuristic % with registry base rates
  9. ContinuumConclusionBuilder — CONCLUSION - THE CONTINUUM answer
  10. Grok-style formatter — @grok observational reply block
  11. Synopsis export available (Markdown, HTML, PDF)

Weights wᵢ are NOT typed by user — ChronofluxWeightConstrual derives them from
scenario context (posed question semantics, region, continuum lean). Field prose
alone does not change weights; explicit scs=N annotations or variable semantics do.

Betting odds and polling are NEVER used.

When signed in and blockchain launched: Calculate triggers PercWalletProvider.creditScenario
→ treasury faucet payout + main-chain block append.


--------------------------------------------------------------------------------
3.3 ANALYSIS TAB — SOCIAL COHESION SCORE (SCS) MODE
--------------------------------------------------------------------------------

Same engine pipeline; output is MarkdownBin-style SSUCF cohesion report

refines PART TWO from party-response SCS signals.

Cohesion report sections mirror PART 1 of this document (vortex/shear/resistance/flow,
baseline, PART TWO refinement, PART THREE interventions, conclusion).

Faucet credit uses SCS as the two-digit outcome (same 50 + outcome cent formula).

3.5 PERCCENT (PERC) WALLET AND BLOCKCHAIN
--------------------------------------------------------------------------------

Currency: Perccent (PERC). Smallest unit: 1 cent = 0.00000001 PERC.
100,000,000 cent = 1 PERC.

Accounts:
  • evolve_treasury — genesis holder; manual send locked after launch
  • evolve_seed_node — internet anchor (hosted on Render)
  • User wallets — register locally; address derived from username + salt

Ledger storage:
  • Local JSON (per device): perc_evolve-chronoflux-principia-chain-1_ledger
  • Seed node persistent disk: chain.json + rendezvous state

Block contents per scenario:
  • Treasury emission tx(s) — per-minute accrual or regeneration
  • Scenario reward tx — faucet payout to user
  • Optional genesis renewal tx — 283M PERC when pool at 1 cent reserve
  • Microblock seal tx — every 100M fair-usage microblocks

Side chain:
  • Each fair-usage event (analysis keystroke activity) records one microblock
  • Chronoflux fingerprint verified per microblock
  • 100,000,000 microblocks → one main-chain seal block
  • Wards bundle 10,000 microblocks each (10,000 wards per seal cycle)

Transfers:
  • Fee: 0.00000001 PERC burned per send
  • Offline receive: pending 12 months, then revert to sender
  • Treasury cannot send manually after launch

Staking:
  • 10% of scenario base (5 cent) per block per holder with confirmed balance

Privacy (on-chain public surfaces):
  • Usernames → 5-character publicAlias (HMAC-SHA256, no reversal)
  • No passwordHash, salt, or password on public GET /perc/ledger
  • IPv4 node endpoints masked as "Private node" on explorer
  • Treasury hidden from public peer listings
